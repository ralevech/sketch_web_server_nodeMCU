Простой и эффективный веб-сервер на базе платы **NodeMCU (ESP8266)**. Проект позволяет подключаться к плате через Wi-Fi и управлять подключенной нагрузкой (например, светодиодами или реле) прямо из браузера на вашем компьютере или смартфоне. ## 💡 О проекте

Этот скетч превращает вашу плату NodeMCU в точку доступа или подключает её к существующей Wi-Fi сети, создавая простой веб-интерфейс. Управление нагрузкой происходит путем перехода по соответствующим ссылкам в браузере.

## ✨ Возможности

*   ✅ Создание полноценного веб-сервера на плате NodeMCU.
*   ✅ Управление цифровыми выводами (включение/выключение нагрузки) через веб-интерфейс.
*   ✅ Простая и понятная HTML-страница управления с кнопками.
*   ✅ Асинхронная обработка запросов (библиотека `ESPAsyncWebServer`) — сервер не тормозит выполнение основной программы.
*   ✅ Безопасное хранение учетных данных Wi-Fi через отдельный файл `secrets.h` (исключен из Git).
*   ✅ Загрузка веб-страницы из файловой системы LittleFS (удобно для сложных интерфейсов).

## 💻 Программное обеспечение

### 1. Среда разработки

Установите одну из версий Arduino IDE:
*   **Официальный сайт:** [Arduino IDE](https://www.arduino.cc/en/software)
*   **Сайт Амперка (русифицированная версия):** [Arduino IDE v.2.3.8](https://amperka.ru/page/arduino-ide?ysclid=mmaqb2rv5h408499504)

**Настройка Arduino IDE для работы с ESP8266:**
1.  Откройте **Файл** -> **Параметры**.
2.  В поле **"Дополнительные ссылки для Менеджера плат"** вставьте строку:
http://arduino.esp8266.com/stable/package_esp8266com_index.json

3.  Перейдите в **Инструменты** -> **Плата** -> **Менеджер плат**.
4.  В поиске введите `esp8266` и найдите пакет **"esp8266 by ESP8266 Community"**. Нажмите кнопку **Установка**.
5.  После установки снова зайдите в **Инструменты** -> **Плата**, прокрутите вниз до раздела **"ESP8266 Boards"** и выберите **"NodeMCU 1.0 (ESP-12E Module)"**.

### 2. Драйвер для соединения USB с платой NodeMCU

Определите, какой чип USB-UART установлен на вашей плате (зависит от производителя), и установите соответствующий драйвер:

*   **CH340G / CH341:**
[Скачать с официального сайта](https://www.wch-ic.com/downloads/CH341SER_EXE.html) |
[GitHub зеркало](https://github.com/SoldierJazz/CH341SER-Driver-For-ch340-ch341.git)
*   **CP2102:**
[Скачать с Silicon Labs](https://www.silabs.com/documents/public/software/CP210x_Universal_Windows_Driver.zip)
*   **CH9102 / CH9102X / CH343:**
[Скачать с WCH](https://www.wch-ic.com/downloads/CH343SER_EXE.html)

### 3. Библиотеки для создания Web сервера

| Библиотека | Описание | Установка |
| :--- | :--- | :--- |
| **[ESPAsyncWebServer](https://github.com/ESP32Async/ESPAsyncWebServer)** | Создает **асинхронный** веб-сервер. Позволяет микроконтроллеру одновременно обслуживать несколько клиентов без зависаний. | Скачайте как ZIP и установите через **Скетч** -> **Подключить библиотеку** -> **Добавить .ZIP библиотеку...** |

### `arduino-littlefs-upload-1.6.3.vsix`

Это плагин для Arduino IDE, необходимый для загрузки веб-страниц и других файлов в память микроконтроллера (LittleFS).

1.  Скачайте файл **`arduino-littlefs-upload-1.6.3.vsix`** со [страницы релизов](https://github.com/earlephilhower/arduino-littlefs-upload/releases/tag/1.6.3).
2.  В Arduino IDE откройте вкладку **Файл** -> **Настройки**.
3.  В поле **"Расположение папки для проектов"** посмотрите путь. Рядом создайте папку `tools` (если её нет) и распакуйте содержимое скачанного `.vsix` (это обычный zip-архив) туда.
4.  Перезапустите Arduino IDE.

**Как пользоваться плагином:**
1.  В папке с вашим скетчем создайте папку `data` (она уже есть в репозитории).
2.  Поместите туда все файлы веб-интерфейса (например, `index.html`).
3.  В Arduino IDE выберите плату (`NodeMCU 1.0`) и правильный COM-порт.
4.  **Закройте монитор порта**, если он открыт.
5.  Нажмите сочетание клавиш **`Ctrl + Shift + P`** (для Windows/Linux), чтобы открыть палитру команд.
6.  Начните вводить **"Upload LittleFS"** и выберите команду **"Upload LittleFS to Pico/ESP8266/ESP32"**.
7.  Дождитесь окончания загрузки.

### 4. Настройка Wi-Fi

Для безопасности ваши логин и пароль от Wi-Fi хранятся в отдельном файле, который не попадает в Git.

1.  Найдите в репозитории файл **`secrets.h-example`**.
2.  Переименуйте его в **`secrets.h`**.
3.  Откройте файл `secrets.h` и введите свои реальные данные:
```c
#define SECRET_SSID "Ваш_Логин_WiFi"
#define SECRET_PASS "Ваш_Пароль_WiFi"
